# Requirements

## Implemented

### Real-time visualisation
High-frequency crowd and temperature updates streamed via WebSocket (Socket.IO) from the Express gateway to the Vue client. The 3D view and dashboard table reflect live sensor data.

### Push server-initiated notifications
Threshold-based alerts (overcrowding, temperature exceeds `maxTemperature`) are published to Redis by the notification service, broadcast to connected clients via Socket.IO, and simultaneously delivered as Web Push notifications to subscribed browsers.

### Digital twin management
Full CRUD for buildings and rooms. Buildings are described by a JSON schema defining room positions and dimensions in 3D space. Rooms can be edited (capacity, temperature threshold, colour, name) after deployment.

### Multi-domain / multi-tenant
The Domain system allows a single CrowdVision instance to serve multiple organisations. Users hold role-based memberships per domain. SSO via OIDC is supported for enterprise identity providers.

## Planned

### Historical data and analytics
Aggregate crowd data by hour/day with charts comparing buildings or floors over time.

### LLM campus assistant
Natural-language queries: "Where is the least crowded study room right now?", "When is the canteen busiest?"

### AI-generated admin reports
Weekly summaries auto-generated by the LLM service from crowd statistics.

### Intelligent push notifications
LLM-generated contextual messages instead of static alert templates.

### Private domains
Invite-only domain access with token-based invite links.
